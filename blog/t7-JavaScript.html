Ruby Hashes vs. JavaScript Objects

Fenuary 7th, 2015

Similarities

Hashes work as key values, and Java-Script objects are just Ruby hashes.

In both languages EVERYTHING is an object.

In Java-Script, objects come in three forms: constants, variables, and methods.

Objects may contain other objects.

All Java-Script objects either contain one constant, or sets of constants working as key/values pairs. (these are variables)

Methods, are declared as variable objects that essential are a 'function' keyword, followed by a pair of parenthesis to optionally encapsulate any parameters the function may have, passed to them by their function call arguments. Then a block of code afterward that acts upon the information given as input (in any), and performs a certain task.

Both take up computer memory

Both may be bypassed by anonymous objects, created in an instant, and gone in an instant.

Both have attributes that store dynamic information and methods that act upon input and/ or the attributes, depending on the nature of the method.

Both may be modified and used dynamically.

Differences

Java-Script objects are directly declared

(in the form: name={}, where name is the name of the object we are creating.), there are no classes in Java-Script, only methods, so you need to directly declare all object attributes upon creation of the object, or dynamically as new attributes are needed.

Java-Script objects automatically come in attr_accessors for all their attributes, so you may reference them directly on default.
